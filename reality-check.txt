## Questions You Should Ask Yourself

Before building anything, honestly answer:

1. **"Will I actually use this?"**
   - If it takes 5 minutes to open and start, you won't use it
   - If it requires manual data entry, will you keep it up?

2. **"Can I build this before my exams?"**
   - If your first exam is in 3 months, don't plan a 6-month project

3. **"Is this solving my actual problem?"**
   - Is your problem: "I don't know what's coming up in exams"? â†’ Build past paper analyser
   - Is your problem: "I forget what I've learnt"? â†’ Build spaced repetition
   - Is your problem: "I don't know what to study"? â†’ Build priority system

4. **"Am I overcomplicating this?"**
   - If you're thinking about databases/APIs/architecture, you're overcomplicating
   - If you're thinking about "What's the simplest Python script that solves this?", you're on track
